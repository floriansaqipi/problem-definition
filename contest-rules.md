# Optimization Contest — Rules

You'll have **4 hours** to solve a complex NP-hard optimization problem. The problem and its instances will be released at the start of the contest.

**Use whatever you want.** Any solver, any library, any language, any tool — open-source, commercial, AI-assisted, hand-written, or anything in between. Run multiple solvers in parallel if you like. We don't care how you reach the solution; we only care that it's valid and competitive.

**Submit one file per instance.** For each problem instance, you may submit at most one solution file. Make sure your submissions arrive before the 4-hour deadline — late files will not be scored.

**Scoring.** Each solution file is scored individually, and your team's final score is the **sum of scores across all instances**. Invalid solutions count as zero for that instance.

**Top 3 follow-up.** The three teams with the highest total scores will be asked to send us their **solvers** (code, scripts, configurations) along with their submitted **output files**, so we can verify the results before final rankings are confirmed.

That's it — good luck.
